# AI統合プラットフォーム

## 背景と目的

社内業務におけるLLM活用を統一的に管理するプラットフォームの構築を目的とする。
従来は各部署が個別にAPI連携を行っていたが、セキュリティとコスト管理の観点から一元化が求められていた。

## 当月の作業内容

- アーキテクチャ設計の完了（マイクロサービス構成）
- API Gateway の選定と検証（Kong vs AWS API Gateway）
- 認証基盤のプロトタイプ実装（OAuth 2.0 + RBAC）
- 負荷テスト計画の策定

## 課題

1. レイテンシ要件（P99 < 200ms）を満たすためのキャッシュ戦略が未確定
2. 社内データの前処理パイプラインとの連携仕様を調整中
