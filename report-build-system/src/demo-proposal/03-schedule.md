# スケジュールと予算

## プロジェクトスケジュール

プロジェクト全体を3フェーズに分割し、段階的に展開する。

### フェーズ1：基盤構築（2026年3月〜5月）

- インフラ環境の構築とGPUサーバーの調達
- LLMの選定とファインチューニング
- ベクトルDBの構築とデータ取り込みパイプラインの実装

### フェーズ2：パイロット運用（2026年6月〜8月）

- 技術部門（50名）を対象としたパイロット運用
- ユーザーフィードバックの収集と改善
- 性能チューニングとセキュリティ強化

### フェーズ3：全社展開（2026年9月〜11月）

- 全部門（500名）への段階的ロールアウト
- 部門別カスタマイズの実施
- 運用体制の確立と引き継ぎ

## 概算予算

| 費目 | 金額 |
|------|------|
| GPUサーバー（4台） | 3,200万円 |
| ソフトウェアライセンス | 800万円 |
| 開発人件費（9ヶ月） | 4,500万円 |
| 外部コンサル費用 | 600万円 |
| **合計** | **9,100万円** |

## リスクと対策

主なリスクとして、GPU供給の遅延、LLMの精度不足、社内データの品質問題が想定される。これらに対しては、複数ベンダーからの並行調達、複数モデルの評価体制、データクレンジングフェーズの設定により対処する。
